等差数列 〜 横へな 2013.6.1 参考問題

問題

数列の中から最も長い等差数列を見つけ出し、その長さを答える。
等差数列というのは Wikipedia - 等差数列 を見ると分かる通り、等間隔に並んだ数の列のことである。
例えば。

  • [1,2,3,4] や [5,6] は間隔が 1 なので等差数列。
  • [4,7] や [11,16,21,26,31] は、間隔が 2 あるいは 5 なので等差数列。
  • [4,6,7,9,10,11,13] や [1,2,3,4,6] は、間隔が1のところと2のところが混じっているので等差数列ではない。

入力と出力

入力は、 12345abcz のようになっている。 各文字は数を示している。文字と値の対応は下表の通り：

文字 0〜9 a〜z
値 0〜9 10〜35
例えば、 012abku は [0, 1, 2, 10, 11, 20, 30] という数列を意味している。この数列に埋まっている等差数列のうち、最長のものは [0, 10, 20, 30] となる。 この等差数列の長さは 4 なので、 4 を出力すればよい。

補足

  • 出力する数値の形式は普通の 10 進数とする。
  • 入力の数列はには同じ数を含まない。
  • 入力の数列は昇順に整列済みである。
  • 長さ1でも等差数列とみなす。
  • 不正な入力 ( 11112222 ( 重複がある ) や 12abcd9 ( 整列済みになっていない ) 、 零壱弐参 ( 0〜9、a〜z に当てはまらない ) など ) に対処する必要はない。
  • 実装ができた方は Qiitaの記事 のコメント欄からリンクを張っていただくと見つけやすくて助かります。
