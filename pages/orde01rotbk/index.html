↓時計回りに90度回転↓

まわせ! Bouwkamp!! 横へな2016.2.6 問題

問題

右図のような感じで、正方形を敷きつめてある。(中の数字は、正方形の一辺の長さ)
敷き詰め方にはルールがあって、

  • 同じ大きさの正方形はない。
  • 隙間なく、重なりもなく敷き詰められている。
  • 正方形の一辺の長さは整数。

となっている。

正方形の並べ方を示すので、その並べ方を 90 度回した結果の一部を出力するプログラムを書け。

入力

入力は 4:(55,44,48)(40,4)(52)(26,29)(23,3)(20,31,21)(5,47)(43)(9,17)(1,8)(32)(25) のような形式。
コロンの後にあるのは、「 Bouwkamp Code 」とよばれるもので、正方形の並べ方を示している。
Bouwkamp Code は

  • 上端の辺がひとつながりになっているものをグループにする。
  • グループ内はコンマ区切り。グループは括弧「()」でまとめられている。
  • グループ内の順序は、x座標順。左から。
  • グループ間の順序は、左上隅の座標順。まずは上から。y座標が同じものは左から。

という具合になっている。図と Bouwkamp Code を見比べるとわかると思う。
カッコ内の数字は、正方形の一辺の長さ。
先程挙げた 4:(55,44,48)(40,4)(52)(26,29)(23,3)(20,31,21)(5,47)(43)(9,17)(1,8)(32)(25) の、コロンの後にあるものは、最初に挙げた図に相当する。
コロンの前の数字は、出力すべきグループの番号を示す。

出力

出力は、入力で与えられた図を時計回りに90度回したものを Bouwkamp コードにしたもの。全部じゃなくて、指定されたグループのみ。括弧も含めて出力すること。
先程の例の場合、コロンの後は
(55,44,48)(40,4)(52)(26,29)(23,3)(20,31,21)(5,47)(43)(9,17)(1,8)(32)(25)
なので、これを 時計回りに 90 度回すと
(43,23,26,55)(20,3)(29)(32,31)(40,44)(1,9,21)(25,8)(17)(5,52,4)(48)(47)
となる。コロンの前は 4 なので、4番目のグループを出力すれば良い。最初のグループが1番目であることに注意。
というわけで、出力すべきは
(32,31)
という文字列になる。

補足

  • 不正な入力に対処する必要はない。
  • サンプルデータにあるデータが現実的な時間で終わるぐらいの計算量になればよい。
  • コロンの前の数字は、出来上がった Bouwkamp Code のグループ数を超えることはないし、ゼロ以下であることもない。
  • Bouwkamp Code は、正方形の敷き詰め問題では一般的なものらしい。私が考えたものではない。
  • 正方形の敷き詰めパターンは、 http://www.squaring.net/sq/ss/spss/spss.html からの引用。
  • 実装ができた方は Qiitaの記事 のコメント欄からリンクを張っていただくと見つけやすくて助かります。
  • この問題は2016年2月6日に開催される オフラインリアルタイムどう書くE01 の問題です。イベントについては Doorkeeper のページ の方を御覧下さい。
